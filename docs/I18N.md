# I18N

## Idiomas suportados

- `pt`
- `en`
- `es`

## Fonte principal

- `client/src/contexts/LanguageContext.tsx`
- `client/src/lib/site.ts`
- `client/src/lib/i18n.ts`

## Como funciona

1. O idioma pode vir de `?lang=`.
2. Se não houver query param, o app tenta `localStorage`.
3. O `LanguageContext` expõe:
   - `language`
   - `setLanguage`
   - `dateLocale`
   - `t`
   - `tm`
   - `formatDate`
4. `applyDocumentLanguage` atualiza o atributo `html[lang]`.

## Estratégias atuais no projeto

- Traduções globais em `client/src/lib/i18n.ts`
- Páginas com `COPY` local tipado
- Dados editoriais mais ricos podem ficar em módulos dedicados por feature

## Padrão recomendado daqui para frente

- Strings globais: `client/src/lib/i18n.ts`
- Strings específicas de feature/página:
  - módulo próprio por página
  - tipado por `SupportedLanguage`
- Dados editoriais complexos:
  - módulos dedicados por domínio
  - sem texto hardcoded direto no JSX

## Regras

- Nunca deixar texto hardcoded em página multilíngue
- Traduzir também:
  - placeholders
  - aria-labels
  - botões
  - FAQ
  - títulos de seção
  - metadata
  - mensagens de erro/loading/empty
- Fallback sempre para PT

## Exemplo recente

As páginas `/utilitarios/horario-mundial/` e `/utilitarios/horario-mercados/` usam:

- `client/src/lib/world-clock-copy.ts`
- `client/src/lib/market-hours-copy.ts`
- `client/src/lib/world-clock-country-details.ts`

Isso evita acoplamento de copy e dataset diretamente no componente.

## Perfis de países no modal

- `client/src/lib/world-clock-country-details.ts` é a fonte única dos perfis.
- O schema resolvido do modal é `CountryProfileContent`.
- PT-BR é a base editorial principal dos perfis curados.
- EN/ES continuam atendidos pelos labels da feature e podem:
  - usar conteúdo curado quando existir
  - cair para PT-BR ou para fallback textual seguro quando a curadoria ainda não foi expandida

## Como adicionar ou enriquecer um país

1. Confirmar que o país existe em `world-clock-countries.ts`.
2. Adicionar ou atualizar o seed em `world-clock-country-details.ts`.
3. Preferir preencher:
   - `editorialSummary`
   - `overview`
   - `quickFacts`
   - `culturalContext`
   - `highlights`
   - `tourism`
4. Se EN/ES ainda não forem escritos, manter PT-BR como base documentada.
5. Validar o modal em PT / EN / ES para garantir que labels e fallback continuam íntegros.

## Checklist antes de subir uma nova página

- título e description mudam por idioma
- canonical continua no path certo
- `hreflang` continua coerente
- headings mudam por idioma
- FAQ muda por idioma
- labels de formulário mudam por idioma
- modal e mensagens de erro mudam por idioma
